# TEST_PLAN.md - Temple Oracle Online

## 1. 測試範疇 (Scope)
- **前端 (Frontend)**: HTML5/CSS3 互動、JavaScript 儀式邏輯、響應式佈局。
- **後端 (Backend)**: Flask API 端點 (擲筊、抽籤、儲存紀錄、功德系統)。
- **資料庫 (Database)**: SQLite 資料持久化、歷史紀錄隔離。

## 2. 測試場景與案例 (Test Scenarios & Cases)

### Scenario 1: 抽籤儀式完整路徑 (Happy Path)
- **TC1.1**: 輸入姓名並選擇問事類別，點擊「點火上香」應跳轉至擲筊畫面。
- **TC1.2**: 點擊「擲筊」，應出現動畫並顯示結果。
- **TC1.3**: 獲得「聖筊」後，應顯示「進入抽籤」按鈕。
- **TC1.4**: 點擊「進入抽籤」，應隨機獲得一首籤詩並顯示完整內容（標題、詩文、故事、解釋）。
- **TC1.5**: 點擊「感謝神明指引」，系統應彈出成功提示並重置流程。

### Scenario 2: 功德與等級系統
- **TC2.1**: 點擊「誠心感恩」，功德點數應增加 1，且顯示對應稱號（初始為「初心信士」）。
- **TC2.2**: 模擬大量功德累積，確認稱號晉升邏輯（>100 -> 虔誠大德, >500 -> 福緣居士）。
- **TC2.3**: 重新整理頁面，功德點數與稱號應保持不變（從 API 重新載入）。
- **TC2.4**: 驗證「重新整理不自動加功德」修復是否有效。

### Scenario 3: 歷史紀錄管理與隔離
- **TC3.1**: 完成抽籤後，歷史紀錄清單應立即出現該筆紀錄。
- **TC3.2**: 開啟無痕視窗或不同瀏覽器，應看到「尚未有紀錄」，確保不同用戶 ID 隔離。
- **TC3.3**: 在無痕視窗操作後，原視窗紀錄不應受到干擾。

### Scenario 4: 邊界與錯誤處理
- **TC4.1**: 未輸入姓名點擊「點火上香」，應跳出提示且不跳轉。
- **TC4.2**: 擲筊結果為「笑筊」或「陰筊」時，不應出現「進入抽籤」按鈕，需重新擲筊。
- **TC4.3**: 模擬後端 API 斷線或資料庫錯誤時的前端提示（選配）。

## 3. 測試環境與工具
- **OS**: Windows
- **Browser**: Chrome (via Antigravity Browser Agent)
- **Backend**: Flask (running on localhost:5000)
- **Database**: SQLite (temple.db)

## 4. 驗收標準 (Acceptance Criteria)
- 所有正向測試案例通過。
- 歷史紀錄與功德數據在重新整理後維持一致。
- UI 視覺符合 Premium Aesthetics 要求（無跑版、動畫順暢）。
